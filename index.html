The hypothesis
We modeled the in-product experience on the CSOM delivery structure. That was a reasonable starting point. But the product has a capability the CSOM doesn't: it can read the workspace in real time.
✅
The CSOM model works. Three goals, structured delivery, expert guidance for complex decisions. This initiative does not challenge that.
🎯
The product's job is different: detect where the customer is, surface only the knowledge that unblocks the next step, and let fast customers move fast — without waiting for a session.
💡
If the product takes on the routine, the CSOM focuses on what humans do best: architectural judgment, stakeholder alignment, complex guidance. Better for everyone.

The integration point
The fastest path to production is not a new surface
Current onboarding UI — what stays, what changes, what the guide replaces.
Replace
3-goal Onboarding Guide widget
Generic, linear, same for every customer. Opens outside the workspace. Exists to support the CSOM's session cadence — the product can do this differently.
→ Replaced by intent-scoped guide panel
Replace
Support Channels widget
Static cards — Community, Academy, Documentation. Reactive: the customer must go find help. Only "Book a session" belongs in onboarding — and it should appear contextually, not permanently.
→ Replaced by contextual CTA (triggers on stuck state)
Amplify
Best Practice Metrics widget
Right signals, wrong context. The tiles show counts with no intent, no prioritization, no next action. Two additions: scope the highlighted metrics to the active intent, add a "How to fix this →" trigger.
→ Passive scorecard becomes active guide trigger
Key integration point
Data Quality Insights bar
Already reads workspace state. Already detects gaps. Already links to filtered views. Two additions: intent-scoping (reorder cards by active goal) + guide trigger on each card. Smallest engineering ask, largest visible payoff.
→ Gap detection becomes the guide entry point
Net new — product gap filled
Persistent guide entry point in nav
Once the onboarding dashboard is closed, there is no obvious way back. A single native nav icon — same position as the EA Assistant — present during the first 90 days or until a maturity threshold is reached.
→ One icon, always available, no dashboard dependency
Unchanged
EA Assistant
Requires AI terms + AI Units — correct to keep separate from the onboarding baseline. The guide is the prerequisite: it fills the workspace with data so the EA Assistant has something real to work with. The upsell is earned.
→ Optional layer, surfaced at intent completion

1

Can we make it available on a subset of new customers?

The pilot runs as a browser extension on a demo workspace today. The question is the right deployment path: native product integration, lightweight SDK, or a controlled rollout mechanism. This is a scoping conversation — no engineering commitment yet.

2

Does it create value?

Measure two signals: does the customer reach their first target report faster than the current average? Does the CSOM spend less time on knowledge delivery in the first four weeks? One cohort answers both. One session of real use beats ten sessions of internal review.
